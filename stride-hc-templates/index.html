1. Pick the device archetype

The archetype determines which scenario library and which compensating controls apply. Not sure which?

Archetype 1 — General-purpose OS legacy

Windows XP/7/Server 2008, older Linux. PACS workstations, imaging modality consoles, lab analysers, pharmacy automation.

Archetype 2 — Embedded RTOS legacy

VxWorks, QNX, ThreadX, proprietary firmware. Infusion pumps, ventilators, patient monitors, anaesthesia, dialysis.

3. STRIDE-HC scenarios

For each category, pick the scenarios that apply to your device. Click chips to toggle — selected chips are coloured. Use the custom-scenarios input to add device-specific ones the library doesn't cover.

4. Generate threat model

Tip: the downloaded files mirror stride-hc-template.md and stride-hc-schema.yaml exactly — drop them into the stride-hc-templates/ directory of your project and they become a regular STRIDE-HC threat model.
